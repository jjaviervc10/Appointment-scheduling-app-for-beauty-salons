---------------------------------------
PASO 2A — CREAR FUNCIÓN buildMiniAppLink
---------------------------------------

En el mismo módulo donde se construyen mensajes (o en un utils/ dedicado),
creá (o reemplazá si ya existe) una función con esta firma:

```ts
interface MiniAppLinkOptions {
  intent: 'booking' | 'reschedule' | 'cancel' | 'availability' | 'services';
  phone?: string;      // E.164, ej: +526143278357
  token?: string;      // token de un solo uso (solo para reschedule/cancel)
  returnTo?: string;   // por defecto 'whatsapp'
  source?: string;     // por defecto 'whatsapp'
}

function buildMiniAppLink(options: MiniAppLinkOptions): string {
  const base = process.env.MINIAPP_BASE_URL || process.env.FRONTEND_URL;
  if (!base) throw new Error('MINIAPP_BASE_URL not configured');

  const routes: Record<string, string> = {
    booking:      '/miniapp/booking',
    reschedule:   '/miniapp/reschedule',
    cancel:       '/miniapp/cancel',
    availability: '/miniapp/booking',    // mismo booking, mismo flow
    services:     '/miniapp/booking',    // mismo booking, misma pantalla de servicios
  };

  const params = new URLSearchParams();
  if (options.phone)    params.set('phone',    options.phone);
  if (options.token)    params.set('token',    options.token);
  params.set('source',   options.source   ?? 'whatsapp');
  params.set('returnTo', options.returnTo ?? 'whatsapp');

  const path = routes[options.intent] ?? '/miniapp/booking';
  const query = params.toString();
  return `${base.replace(/\/$/, '')}${path}?${query}`;
}
```

Reglas:
- `source=whatsapp` siempre para links generados por el bot (tracking futuro).
- `returnTo=whatsapp` para que la mini app muestre el botón "Volver al chat".
- `phone` encodeado automáticamente por URLSearchParams (+→%2B).
- `token` solo cuando sea reschedule o cancel (el token tiene la cita activa).
- No exponer ni logguear tokens en logs de debug.

PASO 2B — REEMPLAZAR BODIES DE MENSAJES
---------------------------------------

En el response builder (archivo encontrado en Fase 1 / paso 3), reemplazá
los textos de respuesta por los siguientes formatos exactos.

IMPORTANTE sobre formato WhatsApp:
- `*texto*` → negrita (SOPORTADO en WhatsApp)
- `_texto_` → cursiva (SOPORTADO)
- Saltos de línea: `\n` (en strings JS)
- Sin HTML. Sin Markdown estándar (# headers no funcionan).
- Máximo 2-3 emojis por mensaje. No más.
- El link siempre después de una línea de CTA, nunca solo.

- Centralizar todos los textos de respuesta en un solo módulo.
