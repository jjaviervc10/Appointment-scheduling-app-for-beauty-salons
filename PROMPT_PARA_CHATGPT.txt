PROMPT PARA ANÁLISIS DE FRONTEND - JAQUELINA LÓPEZ BARBER STUDIO
===============================================================================

Eres un experto en desarrollo frontend con React Native + Expo. Te voy a dar
la información completa de una aplicación móvil de agendamiento de citas para
un salón de belleza/barbería. Necesito que la analices a fondo.

1. RESUMEN DEL PROYECTO
===============================================================================

Nombre: Jaquelina López Barber Studio
Tipo: App móvil de agendamiento de citas con dos roles (dueña/cliente)
Stack: React Native 0.81.5 + Expo SDK 54 + TypeScript strict + expo-router v6
Backend previsto: Supabase (schema: 'booking') — actualmente usa datos mock
Diseño: Premium dark/gold — headers negros, acentos dorados, cards blancos
Estado: Fase 1 completada (frontend completo con mock data)
Repositorio: https://github.com/jjaviervc10/Appointment-scheduling-app-for-beauty-salons

7. SERVICIOS (SERVICE LAYER)
===============================================================================

Todos los servicios actualmente devuelven MOCK DATA. Cada función tiene un
TODO para reemplazar con queries reales a Supabase.

appointments.ts:
  - fetchAppointmentsByDate(date) → AppointmentViewModel[]
  - fetchPendingAppointments() → AppointmentViewModel[]
  - fetchUpcomingAppointments() → AppointmentViewModel[]
  - updateAppointmentStatus(id, status, reason?) → void
  - createAppointmentRequest({clientId, serviceId, startAt, endAt}) → {id}

availability.ts:
  - fetchAvailableSlots(date, serviceId) → TimeSlot[]
  - fetchWeekSummary(weekStartDate) → DaySummary[]
  - fetchTimeBlocks(date) → TimeBlock[]
  - createTimeBlock(block) → {id}
  - fetchWeeklyAvailability() → WeeklyAvailability[] (Lun-Sáb hardcoded)
  - updateWeeklyAvailability(dayOfWeek, startTime, endTime, isActive) → void

incidents.ts:
  - fetchIncidents() → Incident[]
  - fetchActiveIncidents() → Incident[]
  - createIncident(incident) → {id}
  - resolveIncident(id) → void
